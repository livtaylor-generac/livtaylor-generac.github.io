1. **Warning in `KpiResultV3.java` Lines 21-23:**
   ```
   @Builder will ignore the initializing expression entirely. If you want the initializing expression to serve as default, add @Builder.Default. If it is not supposed to be settable during building, make the field final.
   ```
   - **Cause and Solution:**
     - The code uses Lombok's `@Builder` annotation on a class, but some fields have initialization expressions that will be ignored during the building process. 
     - To use these initial values as defaults, you should annotate those fields with `@Builder.Default`.
     - If the initialization values are not meant to be set or changed during the build process, consider making the fields `final`.

2. **Warnings in `FleetSiteMapWithSiteName.java`, `Address.java`, `Audit.java`, `Site.java`, and `RegisterSiteDtoV2.java`:**
   ```
   Generating equals/hashCode implementation but without a call to superclass, even though this class does not extend java.lang.Object. If this is intentional, add '@EqualsAndHashCode(callSuper=false)' to your type.
   ```
   - **Cause and Solution:**
     - These warnings are triggered by Lombok's `@EqualsAndHashCode` annotation. By default, Lombok generates `equals()` and `hashCode()` methods without considering the superclass’s implementation.
     - Ensure whether the superclass’s implementation of `equals()` and `hashCode()` should be invoked.
     - If it's intentional not to call the superclass methods (perhaps because the superclass is not contributing to the identity of the instances), explicitly set `@EqualsAndHashCode(callSuper=false)`.
     - Carefully review if calling the superclass's methods is unnecessary. If you need to include the superclass’s fields in the equality checks, you need to set `@EqualsAndHashCode(callSuper=true)`.

These warnings primarily relate to the Lombok library, which is used to reduce boilerplate code in Java projects. Each suggestion is aimed at ensuring the intended behavior of generated methods and builder patterns, reducing potential bugs and logical errors in object creation and comparison.
